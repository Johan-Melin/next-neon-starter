# Body Tracker — Project Constitution

## What this app does

A responsive body measurement tracker. Users log weight and waist measurements by default, with the ability to add custom measurement types. Data is displayed on a chart and in a paginated table. The app supports imperial/metric unit toggling, dark mode, and authentication.

## Database schema

```typescript
// db/schema.ts — implement exactly this structure
export const users = pgTable('users', {
  id: uuid('id').primaryKey().defaultRandom(),
  email: text('email').notNull().unique(),
  name: text('name'),
  createdAt: timestamp('created_at').defaultNow(),
});

export const measurementTypes = pgTable('measurement_types', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: uuid('user_id').references(() => users.id, { onDelete: 'cascade' }),
  name: text('name').notNull(),        // e.g. "weight", "waist", "chest"
  unit: text('unit').notNull(),         // e.g. "kg", "lbs", "cm", "in"
  isDefault: boolean('is_default').default(false),
  createdAt: timestamp('created_at').defaultNow(),
});

export const measurements = pgTable('measurements', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: uuid('user_id').references(() => users.id, { onDelete: 'cascade' }),
  typeId: uuid('type_id').references(() => measurementTypes.id, { onDelete: 'cascade' }),
  value: numeric('value', { precision: 8, scale: 2 }).notNull(),
  recordedAt: timestamp('recorded_at').defaultNow(),
  notes: text('notes'),
});
```

Seed default measurement types (weight + waist) for every new user on first login.

## Feature spec

### Dashboard
- Line chart showing measurements over time (recharts `<LineChart>`)
- Chart is filterable by measurement type via tabs/dropdown
- Paginated table below chart (10 rows/page) showing: date, type, value, notes
- "Log measurement" button opens a modal/drawer with a form

### Measurement form
- Dropdown to select measurement type (or add new)
- Numeric input with unit displayed inline
- Optional notes textarea
- Date picker (defaults to today)
- Zod validation: value must be positive, max 4 digits before decimal

### Adding custom measurement types
- Settings page with a list of measurement types
- "Add type" form: name + unit (text inputs)
- Delete allowed only if no measurements logged for that type

### Unit toggle (imperial / metric)
- Stored in user preferences in the DB, not just localStorage
- Converting: weight kg↔lbs (×2.20462), waist cm↔in (×0.393701)
- Custom types: user defines the unit, no auto-conversion

### Dropdown menu (top-right avatar/icon)
- Unit toggle: Imperial / Metric
- Dark mode toggle (persists to localStorage + `data-theme` attribute)
- Divider
- Log out

### Auth
- Email + password via Credentials provider
- Optional: Google OAuth (if `GOOGLE_CLIENT_ID` is set, enable it — otherwise skip)
- Redirect to dashboard after login, login page if unauthenticated

## Deployment sequence

Execute in this exact order. Do not proceed to the next step if the current one fails.

### Step 1 — Git setup
```bash
git init
git add .
git commit -m "feat: initial project scaffold"
git tag init          # marks the pre-AI baseline
gh repo create body-tracker --private --source=. --push
```

### Step 2 — Neon database
```bash
# Requires neon CLI: npm install -g neonctl
neonctl auth           # opens browser login
neonctl projects create --name body-tracker --region aws-eu-central-1
# Copy the connection string output into .env.local as DATABASE_URL
neonctl connection-string --pooled   > save to DATABASE_URL
neonctl connection-string --unpooled > save to DATABASE_URL_UNPOOLED
```

### Step 3 — Run migrations
```bash
npx drizzle-kit push
```

### Step 4 — Vercel deployment
```bash
npm install -g vercel
vercel login
vercel link             # links to or creates a Vercel project
# Set all env vars on Vercel:
vercel env add DATABASE_URL production
vercel env add DATABASE_URL_UNPOOLED production
vercel env add NEXTAUTH_SECRET production
# Then deploy:
vercel --prod
```

### Step 5 — Verify
- Visit the deployed URL
- Register a new account
- Log a measurement
- Confirm it appears on the chart and table
- Confirm unit toggle works
- Confirm dark mode toggle works
